## Scope

Reliability, deploy, backup, and incident standards for the deployed Open Brain platform (Hetzner CPX41 host, Coolify, MCP containers, the Command Center app, and the agent runtime). Auditor checks platform work products — deploys, incident records, backup verifications — against these.

## Service-level targets

- **PLT-001:** Platform + MCP container **uptime ≥ 99.5%** monthly (measured by the external pinger + Sentry). — REQUIRED
- **PLT-002:** Incident **acknowledgement** — S1 ≤ 15 min, S2 ≤ 1 h, S3 next business day. — REQUIRED
- **PLT-003:** Backups — Supabase **PITR enabled**; nightly off-box encrypted brain dump; last successful backup **< 24 h** at any check. — REQUIRED
- **PLT-004:** **Restore test passed within the last quarter** (Hermes-run). — REQUIRED
- **PLT-005:** Documented **rebuild** path current; target **RTO ≤ 4 h / RPO ≤ 24 h** (PITR tightens brain RPO). — REQUIRED

## Pass/Fail checks

### Deploys
- **PLT-010:** Deploy is via Coolify from a pinned commit; health check passes before traffic swap. — REQUIRED
- **PLT-011:** A rollback to the prior commit is available and tested. — REQUIRED
- **PLT-012:** No secret values committed; env set in Coolify; names mirror `config/.env.example`. — REQUIRED
- **PLT-013:** The Historian/Researcher network boundary verified after any networking change (`brain-mcp` has no public egress). — REQUIRED

### Security posture
- **PLT-020:** Service-role key server-side only; never shipped to the browser. — REQUIRED
- **PLT-021:** Write endpoints enforce WorkOS role at the data layer. — REQUIRED
- **PLT-022:** Credentials rotated on personnel change / suspected exposure / routine cadence; go-live rotation of dev-shared keys completed. — REQUIRED

### Incidents
- **PLT-030:** Every S1/S2 has a post-incident record written to the One Brain (`evidence`). — REQUIRED
- **PLT-031:** No incident "fix" bypasses the pricing gate or deletes data. — REQUIRED
- **PLT-032:** Repeated failure mode (3× in 90 days) escalated to QC for DMAIC. — REQUIRED

### Brain hygiene (Hermes)
- **PLT-040:** Daily Sort ran and produced a hygiene status; failures alert a human. — REQUIRED
- **PLT-041:** Leaf-folder README coverage 100%; naming/structure conformance at target; brain map + schema catalog current. — RECOMMENDED

## Escalation conditions

- Any gate-bypass possibility, data-loss risk, or boundary violation → immediate S1 + human.
- Backup or restore failure → S2 + human, same day.
- Uptime trending below target two months running → QC review of the single-host SPOF (warm-standby decision).
