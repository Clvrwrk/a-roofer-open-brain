# Standard: Marketing Work Products — v1

QC Owner: Quality Control agent
Effective date: 2026-05-29
Supersedes: none (initial version)
Auditor reference key: standards/marketing/v1.md

---

## Scope

This standard covers all work products produced by `@ob-marketing`, including:

- EEAT publication drafts (case studies, testimonial excerpts, blog posts, photo captions with narrative)
- Schema.org markup blobs attached to published content
- Review request outreach drafts
- Social media content drafts
- Website page copy drafts
- Any content routed through the EEAT flywheel one-click approval flow

---

## Pass/Fail Checks

### EEAT publication draft (all types)

- [ ] **MKT-001:** The source atom(s) for this content have `eeat_signal.publishable_with_consent = true` and `eeat_signal.value ≥ 0.7`. Content derived from atoms below this threshold fails. — REQUIRED
- [ ] **MKT-002:** `consent_flags.publishable_external = true` OR this draft is in the pre-approval queue awaiting client one-click consent. A draft must NOT be published until client consent is recorded. — REQUIRED
- [ ] **MKT-003:** PII clean. The following are prohibited without explicit per-atom homeowner consent: full name (first name alone is acceptable), street address, phone number, email address. — REQUIRED
- [ ] **MKT-004:** No competitive-trade information in published content: no pricing visible to competitors, no subcontractor names, no supplier discount terms. — REQUIRED
- [ ] **MKT-005:** Content is derived from `evidence`-tier or `instruction`-tier atoms only. `inference`-tier atoms may not be presented as fact in external content (they may inform background framing if labeled as estimates or interpretation). — REQUIRED
- [ ] **MKT-006:** No fabricated or embellished claims. Content must be traceable to a specific debrief atom or captured event. "Our crews are the best in the region" without a supporting atom fails. "Our crew worked through a rainstorm to meet the homeowner's move-in date" with a linked debrief atom passes. — REQUIRED
- [ ] **MKT-007:** No profanity, slurs, or language that would embarrass the client. — REQUIRED
- [ ] **MKT-008:** Content length appropriate to format: case study 200–600 words; testimonial excerpt 40–150 words; blog post 400–900 words; social post 50–280 characters. Automated check; Auditor flags but does not auto-fail on length (RECOMMENDED). — RECOMMENDED
- [ ] **MKT-009:** `property_id` or job reference present in the content metadata even if not published in the content text. Traceability must be maintained at the atom level. — REQUIRED

### Schema.org markup

- [ ] **MKT-010:** `LocalBusiness` (or `RoofingContractor` sub-type) schema present on every page. Auditor validates JSON-LD syntax. — REQUIRED
- [ ] **MKT-011:** NAP (Name, Address, Phone) in schema markup matches `config.company.*` values exactly (no abbreviations, consistent formatting). — REQUIRED
- [ ] **MKT-012:** `Review` schema present on any testimonial content. `author` field populated (at minimum with first name or "Homeowner" if client requests anonymity). — REQUIRED
- [ ] **MKT-013:** `Article` schema present on case studies. `datePublished` and `dateModified` in ISO 8601 format. — REQUIRED
- [ ] **MKT-014:** `Person` schema included for named practitioners IF and ONLY IF `original_practitioner.consent_to_attribute = true` on the source atom. — CONDITIONAL
- [ ] **MKT-015:** `Place` schema with full street address included for the property IF AND ONLY IF the source atom has `consent_flags.publishable_external = true` at the property level. City + state alone is acceptable without property-level consent. — CONDITIONAL
- [ ] **MKT-016:** No schema markup references a URL that does not resolve (Auditor performs HEAD check). — REQUIRED

### Review request outreach

- [ ] **MKT-017:** Outreach is sent to the correct homeowner contact (verified against AccuLynx job record). — REQUIRED
- [ ] **MKT-018:** Outreach is sent only after the job is marked `closed` in AccuLynx. No review requests during or before installation. — REQUIRED
- [ ] **MKT-019:** No incentivized review language ("leave us a 5-star review and receive X"). Incentivized reviews violate Google and FTC guidelines. — REQUIRED
- [ ] **MKT-020:** Unsubscribe path present. — REQUIRED
- [ ] **MKT-021:** Sent within `config.marketing.review_request_window_days` (default: 14 days) of job close. Auditor flags late requests but does not fail them (RECOMMENDED). — RECOMMENDED

### Social media content draft

- [ ] **MKT-022:** MKT-001 through MKT-007 pass (source atom quality, consent, PII, fabrication, profanity checks). — REQUIRED
- [ ] **MKT-023:** Platform-specific character / format limits respected (auto-checked). — REQUIRED
- [ ] **MKT-024:** Hashtags, if present, do not include competitor names or any proprietary terms. — REQUIRED
- [ ] **MKT-025:** If the post includes a photo, the photo is confirmed to be a CompanyCam image from the linked job (not a stock photo presented as authentic work). — REQUIRED

### Website page copy

- [ ] **MKT-026:** All factual claims in page copy traceable to an atom or config value. Unsubstantiated superlatives ("the best," "the most trusted") fail unless backed by a specific, citable atom (e.g. a review count or rating with a source). — REQUIRED
- [ ] **MKT-027:** Service area description matches `config.service_area` exactly. No claiming service in counties or zip codes not in config. — REQUIRED
- [ ] **MKT-028:** License number(s) present where required by the jurisdiction's consumer-protection law (`config.jurisdictions.[id].license_display_required`). — CONDITIONAL
- [ ] **MKT-029:** Schema.org markup checks MKT-010 through MKT-016 pass. — REQUIRED

---

## Consent recording checks (cross-cut)

These checks apply across all content types:

- [ ] **MKT-CON-001:** Every published piece has `eeat_signal.consent_recorded_at` stamped with a non-null timestamp. Content may not be published with a null value. — REQUIRED
- [ ] **MKT-CON-002:** The consent record (client one-click approval in Slack, or equivalent) is stored as an atom linked to the content atom. The atom must have `trust_tier = "instruction"` (human-confirmed). — REQUIRED
- [ ] **MKT-CON-003:** If the client selected "Skip" on a content proposal, the source atom has `consent_flags.publishable_external = false`. Auditor verifies this was recorded before the recipe proceeded to any other proposal using the same source atom. — REQUIRED

---

## Escalation conditions

- Any content that names a homeowner with a full last name triggers immediate escalation regardless of other checks.
- Any content that references a property street address without explicit property-level consent triggers immediate escalation.
- Any proposed content where the source atom's EEAT signal cannot be traced to a specific post-op debrief triggers escalation (no free-floating EEAT content).
- Any schema markup that would cause a structured-data error per Google's Rich Results Test escalates to the AM before publish.
