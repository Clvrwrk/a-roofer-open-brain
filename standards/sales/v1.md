# Standard: Sales Work Products — v1

QC Owner: Quality Control agent
Effective date: 2026-05-29
Supersedes: none (initial version)
Auditor reference key: standards/sales/v1.md

---

## Scope

This standard covers all work products produced by `@ob-sales`, including:

- Estimate documents
- Proposals (residential and commercial)
- Scope-of-loss summaries (insurance track)
- Supplement letters
- Canvass route plans and canvass scripts
- Follow-up outreach drafts (to homeowners or commercial contacts)
- Win/loss records

---

## Pass/Fail Checks

### Estimate

- [ ] **SALES-001:** Property address and `property_id` present. — REQUIRED
- [ ] **SALES-002:** Scope of work described in measurable terms: material type, quantity (squares, linear feet), and scope boundary (full replacement vs. partial vs. repair). — REQUIRED
- [ ] **SALES-003:** Manufacturer and product line specified for primary materials (e.g. "GAF Timberline HDZ" not "architectural shingle"). — REQUIRED
- [ ] **SALES-004:** Labor and materials costs itemized separately. A lump-sum estimate without line-item breakdown fails this check. — REQUIRED
- [ ] **SALES-005:** Gross margin is at or above `config.sales.margin_floor_percent` (default: 35%). If margin is below floor, estimate must carry an explicit exception flag reviewed by the AM before delivery. — REQUIRED
- [ ] **SALES-006:** Estimate expiration date present (default: 30 days from issue per `config.sales.estimate_expiry_days`). — REQUIRED
- [ ] **SALES-007:** No material or labor line item derived from memory alone — each must trace to an EagleView measurement, a CompanyCam observation, or a supplier quote. — REQUIRED
- [ ] **SALES-008:** Optional upgrades (ice-and-water shield upgrade, synthetic underlayment, extended warranty) listed as separate line items with their own pricing — never bundled silently into base price. — REQUIRED
- [ ] **SALES-009:** `trust_tier = "evidence"` until homeowner signs the Contingency Agreement or accepts the estimate. — REQUIRED

### Proposal (residential)

- [ ] **SALES-010:** Estimate checks (SALES-001 through SALES-009) all pass as a prerequisite. — REQUIRED
- [ ] **SALES-011:** Payment terms clearly stated: deposit amount (if any), progress draw schedule, final payment trigger. — REQUIRED
- [ ] **SALES-012:** Warranty terms stated per manufacturer's current certification level (GAF Master Elite, OC Platinum Preferred, CertainTeed SELECT ShingleMaster). Warranty tier matches the client's current certification status from `config.manufacturer_certs`. — REQUIRED
- [ ] **SALES-013:** Permit cost line item present IF `jurisdiction.permit_required_for_reroof = true`. — CONDITIONAL
- [ ] **SALES-014:** Scope clarity check: proposal does not use undefined trade terms without explanation (e.g. "tear-off" should be explained as "remove existing shingles down to decking" in a homeowner-facing document). — REQUIRED
- [ ] **SALES-015:** No homeowner PII (beyond name and property address) included without explicit consent. — REQUIRED

### Scope-of-loss summary (insurance track)

- [ ] **SALES-016:** Date of loss matches the storm event atom's `event_date` for the corresponding storm. — REQUIRED
- [ ] **SALES-017:** Affected materials list distinguishes storm-damaged items from pre-existing conditions. Pre-existing items must be explicitly labeled "pre-existing; not part of this claim." — REQUIRED
- [ ] **SALES-018:** Each affected item references photographic evidence (CompanyCam album photo number or URL). A scope item without photo evidence fails this check. — REQUIRED
- [ ] **SALES-019:** ACV (Actual Cash Value) vs. RCV (Replacement Cost Value) distinction present; depreciation holdback amount estimated or noted as pending. — REQUIRED
- [ ] **SALES-020:** Material specifications use current Xactimate pricing database version for the jurisdiction (Researcher provides; version date cited in the document). — REQUIRED
- [ ] **SALES-021:** No items attributed to storm damage that are clearly age-related (granule loss consistent with 20+ year shingle without hail markers). Auditor escalates if scope includes suspect items without photo documentation of storm-specific damage indicators. — REQUIRED

### Supplement letter

- [ ] **SALES-022:** Each supplemented line item has: (a) original adjuster amount, (b) supplemented amount, (c) specific reason (omitted item / under-priced item / code-required item), and (d) source for the correct amount (Xactimate version + date, or code citation). — REQUIRED
- [ ] **SALES-023:** O&P (overhead and profit) line present on supplement where GC markup applies to the job type. — REQUIRED
- [ ] **SALES-024:** Code-required supplement items cite the specific code provision: chapter, section, edition, jurisdiction. — REQUIRED
- [ ] **SALES-025:** Supplement math audited: sum of supplemented items equals stated total supplement request. Any rounding discrepancy > $1.00 fails this check. — REQUIRED
- [ ] **SALES-026:** Supplement letter does not claim items the client's scope-of-loss did not include without a documented reason for the addition. Unapproved scope expansion fails. — REQUIRED

### Canvass route plan

- [ ] **SALES-027:** Affected zip codes trace back to a storm event atom with documented `hail_size_inches` or `wind_mph`. No canvass without a documented storm basis. — REQUIRED
- [ ] **SALES-028:** Warm-call list (prior clients in the area) is segregated from cold-canvass list and flagged accordingly so canvassers know the relationship context. — REQUIRED
- [ ] **SALES-029:** Canvass script does not make claims about storm damage that the canvasser has not personally observed ("your roof was damaged" is prohibited before an inspection; "we're inspecting storm-affected homes in your area" is acceptable). — REQUIRED
- [ ] **SALES-030:** Canvass materials contain the client's contractor license number and the Cleverwork contact for complaints (required by most state canvassing regulations). — REQUIRED

### Follow-up outreach draft

- [ ] **SALES-031:** Homeowner name, property address, and estimate date are accurate (verified against AccuLynx job record). — REQUIRED
- [ ] **SALES-032:** No fabricated urgency ("your roof will fail tomorrow") or false scarcity claims. — REQUIRED
- [ ] **SALES-033:** Unsubscribe path present in any bulk outreach (legal compliance). — CONDITIONAL (required for batch outreach; not required for individual follow-up)
- [ ] **SALES-034:** `trust_tier = "evidence"` on all outreach draft atoms. — REQUIRED

### Win/loss record

- [ ] **SALES-035:** `property_id` and `job_id` (or lead ID if never converted) present. — REQUIRED
- [ ] **SALES-036:** Loss reason categorized from the configured loss-reason taxonomy (config.sales.loss_reasons). If "other" is selected, a free-text explanation is required. — REQUIRED
- [ ] **SALES-037:** Competitor name (if known) recorded separately from loss reason — never embedded in freetext only. — REQUIRED

## Margin floor enforcement

When Auditor detects `gross_margin < config.sales.margin_floor_percent` on an estimate or proposal:

1. Auditor **fails** the artifact with rule SALES-005.
2. The failure message includes the actual margin, the floor, and the delta.
3. `@ob-sales` sends the estimate to the AM with the failure note before the document can be delivered to the homeowner.
4. AM may override with a documented exception (competitive job, strategic relationship, loss-leader intent). If overridden, the exception is written as an atom linked to the estimate.

The margin floor is not a suggestion. It is the only check that also notifies the AM automatically regardless of whether the agent or human catches it first.

---

## Escalation conditions

- Any estimate over `config.sales.large_job_threshold` (default: $75,000) requires AM review before delivery regardless of margin.
- Any supplement letter claiming more than 30% above the insurer's initial estimate requires AM review.
- Any canvass script containing damage-claim language (see SALES-029) escalates immediately to AM; the script is blocked from delivery until reviewed.
