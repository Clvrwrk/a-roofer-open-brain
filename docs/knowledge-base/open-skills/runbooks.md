# Open Skills Runbooks

The public Open Skills runbooks show how the library composes individual skills into durable workflows. They are especially relevant for this repo because a-roofers-open-brain has many repeated workflows that should not depend on a single agent's memory.

## Published Runbooks

| Runbook | Likely Role |
| --- | --- |
| Talk to Published | Turns spoken or rough thought into published output. |
| Release Day | Coordinates a launch/release sequence with checks and communication. |
| Video Production Line | Moves a media project through repeatable production steps. |
| Ship a Page You Can Trust | Combines page implementation, QA, and confidence checks. |
| Research Engine | Structures research into source-backed deliverables. |
| Delegate and Verify | Makes delegation explicit and reviewable. |
| The Flywheel | Converts repeated work into improving reusable assets. |

## Best Fit For This Repo

The strongest immediate matches are:

- Research Engine: use for framework research, vendor evaluation, security model research, and market/SEO intelligence.
- Ship a Page You Can Trust: use for Command Center surfaces, public SSR pages, schema updates, and auth-sensitive pages.
- Delegate and Verify: use with Linear status states so agents can claim, execute, produce receipts, and hand off cleanly.
- Release Day: use for deploys to `cc.proexteriorsus.net`, with preflight, rollback, smoke checks, and stakeholder update.
- The Flywheel: convert repeated successful sessions into repo-native skills and OKF concepts.

## Local Runbook Candidates

This repo should create local runbooks for:

- Command Center auth-sensitive page change
- Supabase schema or RLS change
- Prompt-injection review of external-ingestion feature
- Performance regression triage
- Linear dev-engine task execution
- Incident response and restore drill
