### Architecture decisions

- Open Brain with 5 vertical agents (Accounting, Operations, Sales, Marketing, Executive) + 8 horizontal (Capture, Historian, Researcher, Conductor, Auditor, Quality Control, Innovator, Maintenance).
- Historian internal-only / Researcher external-only is a security boundary; Auditor/QC split keeps standard-setting separate from per-product QA.
- Continuous improvement is two interlocking loops: job learning (inward) and industry research (outward); both share trust tiers, proposal flow, approval matrix, dashboard queues.
- Command Center is the human-in-the-loop approval surface for everything external.
- Memory: curated markdown is source of truth; vector state is rebuildable cache; frozen snapshot at session start; progressive recall Tier 0 → L1 → L2 → L3.
- Imported source projects are evidence, not source code, until sanitized and attributed.

### Key invariants (never violate)

- Never print or commit secrets; `.env.example` is names-only.
- Never expose Supabase service-role credentials to the browser.
- Never index raw archive imports before sanitization.
- Never let Researcher read internal brain memory directly.
- Never let agents self-promote trust tiers, SOPs, skills, or scoring rules — human approval through the operations dashboard.
- No destructive SQL against a client brain; migrations additive and idempotent.
- Never breach memory caps (MEMORY 2,500 / USER 1,375); consolidate, then ask.
- One git worktree per active agent/task; the assigned path is the boundary.
- No AI-initiated external communication, orders, scheduling, or commitments without human approval.
