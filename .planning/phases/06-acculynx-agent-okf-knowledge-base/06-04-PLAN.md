<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| AccuLynx free text → subagent context | Ingested AccuLynx data (job notes, messages, contact names) is external, untrusted input; if the subagent treats it as instructions, that is prompt injection (REQ-09's named risk). |
| subagent tools → repo/live actions | An overly permissive tools list (unscoped Bash/Write) lets a confused or compromised subagent perform destructive repo or write actions. |
| dev subagent → approval authority | The subagent must enqueue only; it must never be able to approve a write (approval is human-only, D-03). |

| T-06-12 | Elevation of Privilege / Tampering | Overly permissive subagent tools list | medium | mitigate | tools list scoped to read/query/enqueue at the doc-verification checkpoint — no unjustified broad Write/Edit or destructive capability (hard rule 1 discipline). |
| T-06-13 | Elevation of Privilege | Subagent gaining approval authority | high | mitigate | Instructions state never-approve (D-03); the subagent enqueues into the Phase 5 human-gated loop only. Verified at the human checkpoint (refuses to approve). |
