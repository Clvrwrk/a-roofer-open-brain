<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser (exec) -> /api/executive/pipeline.json | Untrusted filter query params cross here; the endpoint reads financial/margin data. |
| Untrusted job free-text (AccuLynx) -> rendered DOM | Job names / rep names / lead-source strings originate in AccuLynx and are rendered into the dashboard + drill-down table. |
| Client -> page (WorkOS gate) | The `.astro` page and the `.json` API route are separately reachable HTTP surfaces. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-07-03 | Information Disclosure | /api/executive/pipeline.json | high | mitigate | The route is covered by the existing global WorkOS/service-bearer middleware gate (middleware.ts step 2/4 — unauthenticated /api/* returns buildUnauthorizedResponse). The route MUST NOT be added to PUBLIC_PREFIXES/PUBLIC_EXACT. Verified during Task 1. |
| T-07-04 | Tampering | Filter query params -> Supabase .eq()/.in() | high | mitigate | Allowlist-validate `location`/`region`/`type`/`window` against fixed known sets before building any query filter; reject/ignore unknowns — never pass a raw query-string value into a Supabase filter (Security V5). |
| T-07-05 | Tampering (XSS) | Job/rep/lead-source free text in DOM + drill-down table | high | mitigate | Render all AccuLynx-originating free text via `textContent`/Astro's default HTML-escaping — never `innerHTML` with untrusted strings; the AccuLynx href is built from the numeric/GUID `acculynx_job_id` only, not free text. |
| T-07-06 | Denial of Service (self-inflicted) | Auto-refresh poll amplification | medium | mitigate | Poll cadence is 5 minutes (minutes, not seconds); the JSON endpoint reuses the loader's caching path (same in-memory TTL/inflight-dedup pattern as loadWeeklySnapshot) so concurrent polls don't each re-run the full aggregation. |
</threat_model>
