# Phase 07 ‚Äî UI Design Contract

> Visual and interaction contract for the Executive Sales Pipeline dashboard (`/executive/pipeline`),
> replacing `/weekly-snapshot`. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **This is an EXISTING app with an established look.** The Command Center
> (`app/command-center`) is Astro SSR + vanilla TS ‚Äî no React, no Tailwind, no component
> library. This contract locks the DELTAS this phase introduces (chart styling, filter bar,
> freshness badges, margin-coverage badges, responsive grid) on top of the existing token
> system in `src/styles/global.css`. It does not reinvent the design system.

## Typography

Reuse the existing global type scale exactly ‚Äî do not introduce new sizes/weights.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 15px (`body` base) / 13-14px (table cells, card meta) | 400 (regular; inherited) | 1.5 |
| Label / eyebrow | 11-12px (`.eyebrow`, `.rail-label`, `.live-metric-card span`, badge text) | 700-800 (`font-weight: 700` labels, `800` uppercase eyebrows/badges) | 1.3 |
| Heading (h2 / section head) | 20px (`h2`) | 800 (section titles, e.g. `.snapshot-section-head h3` at 24px is the one outlier ‚Äî reuse only inside a reused snapshot-style block; new sections should use `h2` at 20px) | 1.2 |
| Display (h1 / KPI value) | 28px (`h1`) / 24-30px (`.live-metric-card strong`, `.gap-metric-card span`, `.snapshot-dashboard-head strong`) | 900 (KPI numerals) / 700 (h1) | 1.0-1.15 |

Exactly 2 weights used across new UI: **700 (bold ‚Äî labels, section titles, table header cells)** and **900 (black ‚Äî KPI numerals, the headline margin-% figure)**. Body text stays at inherited 400. (This is 3 effective weights counting the inherited 400 body default, matching the existing app's practice ‚Äî do not add a fourth weight such as 600 semibold.)

Uppercase label convention: `letter-spacing: 0.04em` on all-caps micro-labels (eyebrows, badge text, table `th`), per existing `.live-metric-card span` / `.queue-table th` rules.

## Layout & Composition Contract (phase-specific deltas)

This section captures the decisions the template's generic sections don't have a slot for, but the downstream planner/executor/checker need as prescriptive, not exploratory, guidance.

### Page structure (exec-first hierarchy per D-05/CONTEXT.md)

1. **Global filter bar** (top, sticky under `.top-bar`) ‚Äî location/office, region, commercial-vs-residential selects + the D-03 time-window selector + "Reset filters" ‚Äî built on the existing `.gap-filter-bar` grid pattern (`grid-template-columns: minmax(240px, 1fr) repeat(3, minmax(150px, 190px))`), extended to 4-5 controls. Default state: all locations rolled up, "Last 7 Days" window.
2. **Per-location freshness strip** ‚Äî a compact row of freshness badges (one per active location in the current filter), using the `.live-metric-card`-style tone badges (ready/review/critical), NOT a full metric-card grid ‚Äî these are status chips, not KPIs.
3. **KPI card row** (anchors, D-05 #1-4) ‚Äî `.live-metric-grid`-style 4-up grid (collapses to 2-up at 1180px, 1-up at 820px, matching the existing breakpoints exactly):
   - Pipeline value by stage (funnel proxy ‚Äî stacked horizontal bar, Chart.js)
   - Jobs sold + close rate (with the mandatory snapshot-vs-cohort caption)
   - New leads volume (by location/source)
   - Rep leaderboard + AR rollup (reuses the retired snapshot's leaderboard row list, `.snapshot-row` styling)
4. **Profitability breakdown section** (D-05 #5, the user's emphasized headline lens) ‚Äî margin-%-first cards/table by region, office, commercial/residential, sales team, sales rep. Each breakdown unit is a card showing margin % large (28-30px, weight 900, `--primary`) with the mandatory coverage caption directly beneath it (small, `--text-muted`, never omitted). Job **count** is secondary, smaller, to the side ‚Äî never leads.
5. **Two-level drill-down** (D-09) ‚Äî clicking any KPI/chart segment/breakdown card swaps the sticky right-hand or below-fold panel into a filtered job-level table (reuse `.snapshot-dashboard` / `.snapshot-record-table` pattern), each row linking out via "Open in AccuLynx" to `https://[company].acculynx.com/jobs/{acculynx_job_id}`.

### Freshness badge tone thresholds (Claude's Discretion, D-12)

| Tone | Condition | Visual treatment |
|------|-----------|-------------------|
| `ready` (on-time) | `last_sync_at` within the hourly SLA window | `--secondary-soft` background, no icon change ‚Äî matches `.live-metric-card[data-tone="ready"]` |
| `review` (mildly stale) | `last_sync_at` between 1x and 2x the hourly SLA (i.e., 1-2 hours late) | `--accent-soft` background + amber border ‚Äî matches `[data-tone="review"]` / `.runtime-pill[data-tone="watch"]` |
| `critical` (severely stale) | `last_sync_at` beyond 2x the hourly SLA (2+ hours late) | `--error-surface` background + red border ‚Äî matches `[data-tone="critical"]` |

This reuses the exact 3-tone vocabulary already established by `LiveMetricGrid.astro`'s `data-tone` attribute ‚Äî no new tone vocabulary is introduced.

### Chart styling contract

- Chart.js `responsive: true, maintainAspectRatio: false` on every chart (D-10 ‚Äî required for the KPI-card-stacks-on-mobile behavior).
- Funnel proxy: horizontal stacked bar (`indexAxis: "y"`), stage labels in milestone-normalized casing (`crm_pipeline.current_milestone` values, per RESEARCH.md Pitfall 5 ‚Äî never `acculynx_jobs.current_milestone` Title Case).
- Leaderboard: vertical or horizontal bar, capped at the top 8-10 reps shown by default (avoid a 40+-bar wall ‚Äî "Show all" expands via the same drill-down table, not a longer chart).
- Trend/sparkline (if used for a rep or region mini-trend): thin line, no markers except on hover, `--info` color.
- No 3D effects, no gradient fills on bars, max 5-6 distinct series colors per chart (RESEARCH.md anti-pattern guidance) ‚Äî reuse the semantic palette listed in the Color section, in that fixed order.
- Chart canvas mounts client-side only (`scripts/executive-pipeline.ts`, `DOMContentLoaded`/module-load time) ‚Äî never inside Astro frontmatter (RESEARCH.md Pitfall 6).

### Responsive behavior (D-10, must match existing breakpoints exactly)

| Breakpoint | Behavior |
|------------|----------|
| ‚â•1181px (desktop) | KPI grid 4-up; filter bar single row (5 controls); two-column drill-down layout (report + sticky dashboard panel), matching `.snapshot-layout` |
| 821-1180px (tablet) | KPI grid 2-up (`.live-metric-grid` rule at `@media (max-width: 1180px)`); filter bar 2-up grid; drill-down panel goes static (non-sticky), stacks below the main content ‚Äî matches the existing `.live-workbench`/`.snapshot-layout` collapse at this breakpoint |
| ‚â§820px (mobile) | KPI grid 1-up; filter bar 1-up (stacked full-width controls); charts resize to full container width via `maintainAspectRatio: false`; drill-down job table scrolls horizontally inside its wrapper (`overflow-x: auto`, matching `.queue-table-wrap`/`.snapshot-record-table-wrap`); side rail collapses to the existing icon/label horizontal nav pattern (already handled by `AppShell.astro`, untouched by this phase) |

### Auto-refresh & filter-change UX (D-11, no full-page reload)

- Initial paint: full SSR render (fast, works with JS disabled).
- Filter change: client-side re-fetch against `/api/executive/pipeline.json`, re-render KPI cards/charts/tables in place ‚Äî no navigation, no loading-spinner flash longer than necessary (reuse a small inline loading state on the affected card only, not a full-page overlay).
- Silent poll: `setInterval`, recommended cadence **5 minutes** (Claude's Discretion, D-11) ‚Äî conservative enough to avoid the self-inflicted polling-load pattern flagged in RESEARCH.md's Security Domain table, frequent enough that a left-open exec tab feels "live" given the underlying hourly cron. Poll re-fetches the same JSON endpoint; do not reset the user's active filters or open drill-down on a silent refresh.
