<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Edge Function → AccuLynx API | Outbound GETs cross to a third party using a per-account bearer key; the WRONG key would breach the sandbox-only mandate |
| AccuLynx response → acculynx_api_probe | Untrusted response bodies (containing homeowner PII) cross into stored payload_sample |
| Supabase secret store → Edge runtime | The sandbox key value is resolved at runtime; it must never be logged or persisted |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-prod-probe | Tampering | assertSandbox gate in sweep.ts | mitigate | Code-level hard gate THROWS before any network call unless the resolved secret name is exactly PE_CC_SANDBOX_ACCULYNX_API_KEY; every probe row tagged source_account_key='sandbox'; verify query asserts zero non-sandbox rows (unit test + DB assertion) |
| T-1-pii | Information Disclosure | redactSample / payload_sample | mitigate | Redactor masks homeowner-PII keys to shape tokens before storage; store shapes not values (hard rule 2); unit test proves jobName masking |
| T-1-secret-leak | Information Disclosure | key resolution + logging | mitigate | Only the secret NAME constant in source; resolved key value never logged or written to a row; .env gitignored |
| T-1-ratelimit-ban | Denial of Service (self) | sweep pacing | mitigate | Pace less than or equal to 8 req/s on the single sandbox key; 429 backoff with Retry-After + exponential + jitter; 120s budget with watermark resume |
| T-1-sparse-seed | (data-quality) | Tier B/C ID chaining | accept | If sandbox lacks seed data, record 'empty' distinctly and note Phase-4 write-seeding — not a security threat, documented as a known gap (A1) |
| T-{phase}-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs; Deno std-lib fetch + Deno.env only (RESEARCH §Package Legitimacy Audit: N/A) |
