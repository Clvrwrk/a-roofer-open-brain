## Column contract (grounded in 182 shape + RESEARCH.md)

**`acculynx_pending_write`** (mirrors 182's idempotent DDL + CHECK-enum pattern):
- `id integer generated always as identity primary key`
- `work_key text not null unique` — join key to dashboard/LiveWorkItem
- `lane text not null check (lane in (…the 17 lane names…))`
- `target_env text not null default 'sandbox' check (target_env in ('sandbox','prod'))`
- `account_key text not null` — the single AccuLynx account this write targets
- `endpoint text not null` — resolved method+path for human-readable preview
- `payload jsonb not null` — the request payload (redacted before store)
- `dry_run_render jsonb` — the built request object shown to the approver
- `idempotency_key text not null unique`
- `status text not null default 'pending_review' check (status in ('pending_review','approved','executed','rejected','failed'))`
- `approver text` — set at decision time
- `exec_result jsonb` — set at execute time
- `department text` — which department surface this maps to (from the lane)
- `created_by text` — the enqueuing agent actor
- `created_at timestamptz not null default now()`, `updated_at timestamptz not null default now()`

**`acculynx_write_action_log`** (audit shape referenced from atom_access_log's columns, NOT reusing that table):
- `id integer generated always as identity primary key`
- `pending_write_id integer references public.acculynx_pending_write(id)`
- `work_key text`
- `lane text not null`
- `target_env text not null`
- `account_key text not null`
- `idempotency_key text not null`
- `actor text not null` — approver/actor who caused the execute
- `request_sample jsonb` — redacted outbound request
- `response_sample jsonb` — redacted AccuLynx response
- `http_status integer`
- `outcome text not null check (outcome in ('executed','failed'))`
- `created_at timestamptz not null default now()`

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Astro service-role client → acculynx_pending_write | The Command Center tier writes/reads pending-write rows via the service-role key |
| edge function → acculynx_write_action_log | The write-action function inserts immutable audit rows |
| consent domain (atom_access_log) ↔ write-action domain | Two DISTINCT domains that must not be conflated (hard rule 6) |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-05-07 | Tampering | idempotency_key uniqueness | high | mitigate | `idempotency_key` is a UNIQUE index — a duplicate execute cannot create a second row, backstopping the edge function's double-fire guard at the DB layer. |
| T-05-08 | Repudiation | acculynx_write_action_log | high | mitigate | Immutable audit table (service-role insert only, no update/delete grant) records every executed write with actor + target + redacted request/response — SC2 audit trail. |
| T-05-09 | Information Disclosure | consent-domain confusion | high | mitigate | Write-action audit is a NEW table; `atom_access_log` (consent domain, hard rule 6) is neither reused nor extended — column shape referenced only. RLS enabled on both new tables. |
| T-05-10 | Tampering | destructive migration | high | mitigate | DDL is additive/idempotent only (create-if-not-exists, no DROP/TRUNCATE/retype); verify-gate greps confirm no destructive statements (hard rule 1). |
| T-05-11 | Elevation | over-broad grants | medium | mitigate | Both tables `service_role`-scoped; no blanket `to public`/anon grant; RLS enabled by default. |
</threat_model>
