Build the `acculynx-write-action` Supabase Edge Function as a pure, unit-tested request-builder core (`action.ts`) plus a thin HTTP entrypoint (`index.ts`), mirroring the `acculynx-write-sweep` structure. ## The 17 proven-safe lanes (D-06 — authoritative, from docs/37 + write-capability.md)

**Writable (12):**
1. `POST /contacts` — creates a contact (dependency-root seed); `mailingAddress` state/country are OBJECTS
2. `POST /jobs` — creates an UNASSIGNED lead; `jobCategory.id` is Int32 (use `intId`); `locationAddress` state/country are STRINGS
3. `POST /jobs/{jobId}/payments/received`
4. `POST /jobs/{jobId}/payments/expense`
5. `PUT /jobs/{jobId}/address` — state/country are STRINGS (use `buildJobAddress`)
6. `PUT /jobs/{jobId}/initial-appointment`
7. `PUT /jobs/{jobId}/insurance`
8. `PUT /jobs/{jobId}/insurance/insurance-company`
9. `PUT /jobs/{jobId}/lead-source`
10. `PUT /jobs/{jobId}/priority`
11. `DELETE /jobs/{jobId}/representatives/ar-owner` (idempotent)
12. `DELETE /jobs/{jobId}/representatives/sales-owner` (idempotent)

**Write-only (5) — no follow-up GET (Pitfall 4):**
13. `POST /financials/{financialsId}/worksheet/items`
14. `POST /jobs/{jobId}/messages`
15. `POST /jobs/{jobId}/photos-videos` (multipart/form-data, 202)
16. `POST /jobs/{jobId}/representatives/company`
17. `POST /jobs/external-references` (idempotency anchor)

NOT in scope (do NOT build wrappers): `PUT /jobs/{id}/custom-fields*` (blocked-by-dependency, dropped per amended D-04, Pitfall 5), the 2 fragile lanes (`PUT /jobs/{id}/trade-types`, `DELETE /jobs/{id}/initial-appointment`, Pitfall 6), and the 17 blocked-by-dependency lanes.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| decision endpoint → edge function | Command Center (WorkOS/bearer authed) invokes the edge function with service-role auth; untrusted lane/payload/targetEnv cross here |
| edge function → AccuLynx API | The function holds AccuLynx keys; a wrong account_key or targetEnv here writes to the wrong (possibly live) account |
| agent-authored payload → request builder | Payload originates from an agent; malformed/malicious fields must be validated before an AccuLynx request is built |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-05-01 | Tampering / Elevation | assertTarget (edge) | high | mitigate | `assertTarget(targetEnv, accountKey)` throws before any resolution/network call unless targetEnv is exactly sandbox (default) or prod with a named account — prod is never implicit (D-09 barrier #1). Unit-tested in Task 2. |
| T-05-02 | Tampering / Repudiation | idempotency guard (execute path) | high | mitigate | `computeIdempotencyKey` + a pre-execute check against `acculynx_write_action_log` for a prior success short-circuits a duplicate write (esp. payment double-post, D-05). |
| T-05-03 | Tampering | per-request key resolution (index.ts) | high | mitigate | Key resolved fresh per-request from `acculynx_accounts` by the requested `account_key` via `resolveKey`; never a cached/module-level key — prevents a sandbox payload firing on a prod key or vice versa (RESEARCH Pattern 3). |
| T-05-04 | Information Disclosure | request/response persistence | high | mitigate | `redactSample()` applied to every outbound request body and inbound AccuLynx response before it is returned in a preview or written to the audit log (hard rule 2). AccuLynx key never echoed. |
| T-05-05 | Tampering (injection) | buildWriteRequest payload | medium | mitigate | Per-lane V5 input validation: each lane validates required payload fields and throws a field-naming error before constructing the AccuLynx request; strict enums pass through only known values. |
| T-05-06 | Tampering | write-only lane verification | low | accept | Write-only lanes have no read-back path; the 2xx + echoed body persisted to the audit log is accepted as the only evidence (Pitfall 4, documented AccuLynx guardrail). |
</threat_model>
