| Field | Why Innovator reads it |

**Step 3 â€” Apply `skills/infranodus/critical-perspective` to the proposed A3 draft**

Self-critique questions generated by Critical Perspective:
- *"What if @ob-sales already has a manual workflow that is faster than any automated version because they use judgment on which photos are most persuasive to adjusters?"*
- *"Does automating photo-linking actually reduce the rework rate, or does it shift the failure mode to incorrect photo selection?"*
- *"What happens when CompanyCam is down? Does the automated link fail silently or loudly?"*

These questions become Section 6 (Risks) and Section 7 (Alternative considered) of the A3.

**Step 4 â€” Apply `skills/infranodus/shifting-perspective` to the proposed skill**

Frames evaluated:
- *Foreman's view:* Does photo-linking feel like a burden today? (Yes â€” manual task after a full install day.)
- *Estimator's view:* Does the photo selection matter for claim success? (Yes â€” wrong photos reduce acceptance rate.)
- *Insurance adjuster's view:* Do they care about photo presentation or just completeness? (Completeness first; presentation second.)
- *Client's view:* Does a faster supplement matter to them? (Yes â€” cash flow on ACV vs. RCV timing.)

Shifting perspective confirms the proposal; the estimator's concern about photo selection becomes a constraint on the skill design (the skill must surface candidate photos for human confirmation, not auto-select).

## Quarterly Resurrection Review Flow

```json
{
  "rpc": "get_proposal_archive",
  "args": { "client_id": "self", "status": ["killed", "deferred"] }
}
```

For each archived proposal, Innovator checks:
1. Has the occurrence rate changed?
2. Has the model cost changed (affecting agent operation cost)?
3. Has the error cost increased (e.g., a claim rejection that was foreseeable)?

If any condition has materially changed: write a flag atom and re-route to Chris/AM. If unchanged: no action.

## Failure Handling

| Failure | Behavior |
|---|---|
| A3 math does not clear 10x gate | Write the honest A3 with actual numbers; recommend defer or kill; never inflate the math |
| `search_thoughts` returns insufficient data for baseline measurement | Note in A3 Â§1: "baseline unmeasured â€” minimum [N] occurrences with timing data needed before proceeding"; recommend defer with a measurement collection request to Conductor |
| `write_proposal` fails (file write error) | Retry once; if still failing, surface to Conductor; do not route A3 notification until the file is written |
| `skills/infranodus/critical-perspective` unavailable | Proceed with manual self-critique; note in A3 that InfraNodus skills were unavailable for this review; flag to Conductor |
| `skills/infranodus/shifting-perspective` unavailable | Same as above â€” proceed manually; note in A3 |
| External Researcher scan returns no technology signals of relevance | Log "no signals this week" in weekly Innovator status atom; no A3 action required |
| Deferred proposal's `revisit_at` date arrives but Conductor does not deliver the alert | Quarterly resurrection review will catch it; interim: Innovator logs deferred proposals with a self-check on the relevant weekly run |
