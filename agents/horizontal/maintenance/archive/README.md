# Maintenance Playbook Archive

This directory holds all prior versions of `PLAYBOOK.md`.

Every time a Kaizen Review cycle produces an accepted change, the current version of `PLAYBOOK.md` is copied here before the new version takes effect. The archive file is named `PLAYBOOK-v[X.Y].md`, where `[X.Y]` is the version being retired.

---

## Why this directory exists

The Maintenance agent is bound by its own core rule: **never delete history**. That rule applies to the playbook itself. A playbook that has been superseded is not a mistake to be erased — it is a record of what the standard was at a given time, why it was adequate for that phase of brain operation, and why it changed.

Future practitioners (or future agents) reviewing this archive can answer the question: *"When did Maintenance start handling consent-expiration atoms? What triggered that change? What problem were we solving?"* The answer lives in the archived playbook version and the kaizen observation that motivated the change.

---

## Naming convention

```
PLAYBOOK-v1.0.md   ← initial release
PLAYBOOK-v1.1.md   ← first accepted Kaizen change
PLAYBOOK-v1.2.md   ← second accepted Kaizen change
...
PLAYBOOK-v2.0.md   ← major structural change (requires explicit Chris approval)
```

Minor versions (v1.x): accepted Kaizen changes from the monthly review cycle.
Major versions (v2.x): structural changes — adding a new 5S phase, changing the cadence pattern, splitting into sub-playbooks, or any change that requires explicit Chris approval beyond the standard monthly review.

---

## Access and policy

- These files are **read-only** once archived. No agent or human should modify an archived version.
- These files are **not subject to the cold-archive Sustain process**. Playbook archive files are retained indefinitely, regardless of retrieval frequency.
- If a future Kaizen Review determines that a prior playbook change was a mistake and should be rolled back, the rollback is implemented as a *new* version of `PLAYBOOK.md` that restores the prior behavior — not as an edit to the archived file.
